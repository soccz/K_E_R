# DART 기업 종합 진단 프레임

## 1. 분석 목적

이 시스템은 **투자 판단(매수/매도)이 아닌 기업 자체의 상태 진단**을 만든다. 의사가 환자 풀 바디 체크업을 하듯, 사업·재무·수익성·자본배분·업황·경쟁·거버넌스를 부위별로 본다. 결론보다 *각 부위의 상태와 추세*가 중심이다.

## 2. 독자 가정

비전문가도 이해할 수 있어야 한다. 금융 지식이 거의 없는 독자가 보고서를 읽어도 회사 상태를 따라올 수 있어야 한다. 이게 모든 작성 결정의 1순위 기준이다.

## 3. 출처·추론 규칙 (절대 규칙 — 위반 금지)

이게 보고서 신뢰성의 1순위 기준. 접근성 5규칙보다도 위.

### 3.1 출처 엄격주의
- **모든 사실 주장에는 검증 가능한 출처가 명시되어야 한다.**
- 출처가 *없거나*, *애매하거나*, *조금이라도 모호하면* — 그 주장은 **언급하지 말고 통째로 제외**한다.
- "보도에 따르면" / "업계에 따르면" / "~로 알려져 있다" / "관계자에 의하면" 식의 출처가 모호한 표현은 **금지**.
- 검증되지 않은 풍문·소문·미확인 언론 보도는 *없는 것처럼* 처리.
- 회의적 → 침묵: 의심스러우면 그 문장을 빼라. 채우려고 약한 출처를 끌어다 쓰지 말 것.

### 3.2 출처 인용 형식
- **DART 공시**: `[공시명, 접수일, 회사명]` 예: `[1Q 잠정실적, 2026-04-08, 삼성전자]`
- **재무제표**: `[XBRL fnlttSinglAcntAll, 사업보고서 2025, 항목명]` 예: `[XBRL, 2025사업보고서, 영업활동현금흐름]`
- **시장 데이터**: `[출처, 조회일자]` 예: `[KRX, 2026-05-02 종가]`
- **법규·규정**: `[법령명, 조항]` 예: `[자본시장법 제161조]`
- **외부 자료**: 가능한 경우 직접 링크 또는 명확한 식별자(URL·DOI·문서번호)
- 인용은 인라인 각주 형태 또는 문단 끝 메타 라인. 가독성을 위해 너무 자주 끊지 말되 *생략은 금지*.

### 3.3 추론 명시 규칙
원자료에서 직접 읽을 수 없는 모든 *해석·추정·전망*은 추론임을 명시한다.

- 인라인 마커: `*(추론)*` 또는 `[추론]`
- 예시: 
  - 사실: "삼성전자 1Q 영업이익 6.6조원 *(잠정실적, 2026-04-08)*"
  - 추론: "메모리 가격 회복 모멘텀이 2Q에도 이어질 가능성 *(추론 — 1Q 출하량 증가율과 컨퍼런스콜 톤에서 유추)*"
- **추론의 근거를 함께 적는다** — "왜 그렇게 추론했는가"를 한 줄이라도. 근거 없는 추론은 추측, 추측은 금지.
- 추론이 사실보다 길어지면 그 단락 자체를 재고 — 보고서가 추측 위에 서면 안 됨.

### 3.4 "확인되지 않음" 표준 처리
*침묵*할 것과 *명시적으로 확인되지 않음*으로 남길 것을 구분한다.

- **침묵**: 그 항목이 그 보고서에서 분석 가치가 없거나 무관할 때 → 그냥 안 쓴다
- **확인되지 않음**: 페르소나·프레임이 *반드시* 보려고 한 항목인데 출처에서 잡히지 않을 때 → 명시한다

각 섹션 끝에 *해당될 경우* 다음 박스를 표시:

```
> **확인되지 않음**
> - 미국 매출 비중 (회사 공시는 지역별이 아닌 사업부별로만 분해)
> - 장기 외국인 유형별 보유 추세 (DART 임원/주요주주 공시에서는 잡히지 않음, 별도 데이터 필요)
```

이렇게 두면 (1) 감사 추적이 가능하고 (2) 다음 분기에 동일 항목이 잡히는지 비교할 수 있고 (3) 모호한 출처로 채우려는 유혹을 차단함.

"확인되지 않음"이 한 보고서에 여러 번 등장해도 OK — 그게 정직함이다.

### 3.5 출처 우선순위 (강제 적용)

같은 정보가 여러 출처에 있을 때, *반드시 상위 등급 출처를 사용*한다. 하위 등급은 보완·트리거로만.

| 등급 | 종류 | 사용 가능 범위 |
|---|---|---|
| **1순위 (사실 근거)** | DART 원공시, XBRL 재무제표, KRX 시세, 회사 IR 원문 (보도자료·실적발표자료) | 모든 사실 주장의 근거로 사용 가능 |
| **2순위 (사실 근거 보조)** | 법령·시행령, 금감원·한국은행·통계청·산업부 공식 통계, 거래소 공시 시스템 | 1순위가 다루지 않는 영역에서 사실 근거로 사용 가능 |
| **3순위 (참고·트리거)** | 언론 기사, 증권사 리포트, 업계 매체, 컨퍼런스콜 보도 | **사실 근거로 사용 금지.** "이런 시각도 있다" 정도의 *참고*나, 다음 검증을 트리거하는 정도로만 |

규칙:
- 3순위에서 본 정보는 **반드시 1·2순위에서 재확인**해야 보고서에 사실로 들어갈 수 있다
- 3순위만 출처일 때는 *추론* 마커를 달거나, 통째로 빼야 한다
- "언론에 따르면 ~다"는 절대 금지. "[언론사, 날짜] 기사에서 X 가능성이 거론됨 — 1순위 출처에서 확인되지 않음"이 정직한 표현

### 3.6 데이터 기준시점 의무 (모든 보고서 상단)

모든 보고서(`00_종합진단.md`, 산업노트)는 본문 시작 직전에 **기준시점 박스**를 둔다. 나중에 숫자가 달라졌을 때 추적 가능하게.

```
> **데이터 기준시점**
> - 작성일: 2026-05-02
> - DART 조회: 2026-05-02 21:10 KST (최신 공시까지 반영)
> - 재무제표: 2025 사업보고서 + 2026 1Q 잠정실적
> - 시장가격: KRX 2026-05-02 종가
> - 외인 동향: KRX 2026-05-02 외국인 보유잔고
> - 환율·금리: 한국은행 ECOS 2026-05-02
```

이 박스는 합본 헤드라인 바로 아래, 본문 시작 전에 위치. 모든 sub-agent는 이 데이터 기준시점을 헤더에서 받아 일관되게 인용한다 — 같은 보고서 안에서 시점이 흩어지면 안 됨.

## 3a. 접근성 5규칙 (모든 섹션 의무)

1. **30초 회사 소개를 매 보고서 첫머리에 반복** — 사업이 무엇이고 사이즈가 어느 정도이고 어떤 비유가 어울리는지. 반복되어도 OK, 일관성 자체가 가치.
2. **모든 숫자는 비교군과 함께** — "ROE 12%" 단독 금지. "ROE 12% (동종업계 평균 9%, 작년 14%)" 형태. 비교군 숫자도 출처 명시.
3. **전문용어 첫 등장 시 인라인 한 줄 풀이** + 보고서 끝 `10_용어사전.md`에 자동 누적.
4. **전문 결론은 평이한 한국어로 한 번 더** — "자본효율 악화"라고 썼으면 다음 문장에 "버는 돈 대비 투자 효율이 나빠지고 있다는 뜻이다"를 덧붙인다.
5. **각 섹션마다 비유 한 줄 의무** — 어렵게 느껴지는 개념은 일상 비유로 마무리.

## 4. 분석 가중치

- **가치/펀더멘털 50%** — 사업구조·자본배분·수익성·경쟁우위
- **매크로/사이클 30%** — 업황 사이클·외부변수 노출·외인 자금 동향·지정학
- **회계/공시 디테일 20%** — 주석·우발채무·거버넌스 신호

동적 조정:
- 감사보고서 시즌(3월말) → 회계 비중 ↑ (30%까지)
- 주요 경기 변곡점·금리 전환기 → 매크로 비중 ↑ (40%까지)
- 사업 구조개편 시점(분할·합병·스핀오프) → 가치 비중 ↑ (60%까지)
- 지정학 격변기(미중 갈등·전쟁 위협 고조) → 매크로 비중 ↑ (45%까지)

## 5. 평일판 — 기업 종합 진단 보고서

### 5.1 폴더 구조

```
companies/<기업명>/<회계연도>-<주기>/
```
예: `companies/삼성전자/2026-Q1/`, `companies/현대차/2025-annual/`

### 5.2 섹션 파일 (각 2~5 페이지)

#### `01_사업구조진단.md`
- 무엇으로 돈을 버는가? 세그먼트별 매출·이익 비중
- 직전 분기 / 1년 전 대비 비중 변화
- 새로 시작한 사업, 접은 사업, 비중 이동
- 사업 다각화의 정도와 방향성
- *비유 한 줄 의무*

#### `02_재무건강진단.md`
- 자본구조: 자기자본 vs 부채, 차입금 만기 분포
- 현금흐름 3종(영업/투자/재무)의 패턴, free cash flow
- 운전자본: 매출채권·재고·매입채무 회전 추세
- 단기 유동성: 현금성자산 vs 단기부채
- *비유 한 줄 의무*

#### `03_수익성진단.md`
- 매출총이익률 / 영업이익률 / 순이익률 추이 (직전 4~8분기 시계열)
- ROE / ROIC — 절대값과 추세, 동종업계 비교
- 이익률 변화의 분해: 가격 효과? 물량 효과? 비용 효과? 믹스 효과?
- *비유 한 줄 의무*

#### `04_자본활용진단.md`
- capex 규모와 방향 — 어떤 사업·지역에 투자?
- M&A — 최근 인수/매각, 그 의도
- 주주환원: 자사주 매입·소각, 배당 추세
- 자본 활용 효율: ROIC vs WACC 비교
- *비유 한 줄 의무*

#### `05_업황과사이클진단.md`
- 이 산업이 지금 사이클의 어느 지점? (회복기/호황/둔화기/침체기)
- 회사가 사이클 대비 잘하고 있나 못하고 있나?
- 외부변수 노출도: 환율·금리·원자재·글로벌 수요
- 다음 분기/연간 외부 변수 전망
- *비유 한 줄 의무*

#### `06_경쟁포지션진단.md`
- 시장점유율 추세
- 직접 경쟁자와 재무지표·전략 비교
- 진입장벽·해자(moat)가 무엇인가? 강해지나 약해지나?
- 신규 위협: 신기술·신규 진입자·규제
- *비유 한 줄 의무*

#### `07_거버넌스리스크진단.md`
- 지배구조: 최대주주·우호지분, 변동 사항
- 주석에서 눈에 띈 항목: 특수관계자 거래·우발채무·회계정책 변경
- 임원 변동, 이사회 구성 변경
- 잠재적 미래 리스크: 소송·규제·환경 이슈
- *비유 한 줄 의무*

#### `08_이번분기변화.md`
- 직전 분기 보고서 대비 *새로 바뀐 것들만* 모은 섹션
- 01~07 각 섹션의 변화점을 한 곳에 모음
- "이번 분기 헤드라인" 3~5개로 마무리

#### `09_추적사항.md`
- 다음 분기 보고서가 나오면 이거 변했는지 보겠다 — 5~7개 항목
- 우려 사항 (예: 부채 증가 → 다음 분기까지 줄어드나?)
- 기대 사항 (예: 신규 capex의 매출 기여 시작 시점)

#### `10_용어사전.md`
- 이번 보고서에서 등장한 모든 전문용어의 풀이
- 본문에서 인라인으로 첫 등장 풀이를 한 번 더 모은 것

### 5.3 합본 — `00_종합진단.md` (최종 deliverable)

이게 사용자가 실제로 읽는 파일이다. 구조:

```
1. 헤드라인 (1줄 요약)
2. 데이터 기준시점 박스 (섹션 3.6 형식 의무)

7. 종합 한 페이지 — owner-valuation 톤 ("3년 후에도 매력 있는가, 시총 X조원에 통째로 살 만한가") + 5~7줄 진단
```

분량 목표: 20~30 페이지.

## 6. 주말판 — 산업노트

### 6.1 위치
```
industry_notes/<YYYY-WNN>-<섹터>.md
```
예: `industry_notes/2026-W18-반도체.md`

### 6.2 단일 파일 구조 (5~8 페이지)

```
1. 한 줄 요약
2. 데이터 기준시점 박스 (섹션 3.6 형식 의무)
3. 30초 산업 소개
4. 이번 주 흥미로운 이유 (트리거 — 무슨 공시·이벤트가 있었나)
5. 워치리스트 안의 종목들 — 한 줄씩 비교 표
6. 종목 간 차이를 만드는 변수 3개 (이게 핵심)
7. 사이클 지점 — 외부 변수 영향
8. 비유로 이해하기
9. 다음 변곡점 (이벤트·공시 캘린더)
```

### 6.3 섹터 트리거 우선순위

가뭄 주(공시 거의 없는 주)에 어떤 섹터를 다룰지 결정하는 우선순위:
1. 가장 오래 다루지 않은 섹터
2. 워치리스트 종목들의 직전 공시에서 나온 *외부 변수 변화 시그널*이 가장 큰 섹터
3. 매크로 헤드라인이 그 주에 가장 강했던 섹터 (금리·환율·원자재 등)

## 10. Sub-agent에 대한 참고

각 섹션을 생성하는 sub-agent는:
- 이 `_frame.md`와 **`_persona.md`를 함께** 시스템 프롬프트로 받는다
  - `_frame.md` = "어떻게 쓸지" (구조·규칙·가중치)
  - `_persona.md` = "누구의 눈으로 볼지" (사용자 투자 철학·적신호·호재 우선순위)
  - **두 문서가 충돌하면 페르소나 우선**
- **출처·추론 규칙(섹션 3)은 절대 규칙** — 출처 없거나 모호하면 그 주장은 통째로 제외, 추론은 명시. 이걸 위반하느니 섹션이 짧아지는 게 낫다.
- 자신이 담당하는 섹션 번호만 작성한다 (다른 섹션 침범 금지)
- 같은 데이터를 받지만 자기 섹션의 관점만 본다
- 접근성 5규칙을 모두 적용한다 — 특히 비유 한 줄 의무를 빼먹지 않는다
- 페르소나의 적신호·호재 표에서 *자기 섹션이 담당하는* 항목들의 우선순위(★)에 따라 분량을 조절한다
- 섹션 끝에 "이번 회 등장 용어 [...]" 메타 라인을 넣어 합본 시 `10_용어사전.md`에 자동 수합되게 한다

## 11. 페르소나 기반 섹션 변형

`_persona.md`로 인해 각 섹션은 일반 분석 프레임에 다음을 추가로 다룬다:

- **01_사업구조진단**: 매출의 해외 비중·수출 노출도, 미국 시장 노출도, 정부 정책 의존도, *국내 개미·정책 테마주 의존도*
- **02_재무건강진단**: 영업이익 vs 영업CF 괴리를 *우선* 점검
- **03_수익성진단**: 영업CF 기준 수익성을 회계 이익보다 우선
- **04_자본활용진단**: capex가 미국 트렌드 정합한 방향인지 평가
- **05_업황과사이클진단**: 환율·유가(섹터)·미국 금리·외국인 매매·*지정학 리스크* 5종 매크로 의무
- **06_경쟁포지션진단**: **3년 단위** 기술 트랙 일관성, 미국 시장 검증 여부, *압도적 기술 보유 종목 한정 — 미국 자본의 인수·IP 시나리오*
- **07_거버넌스리스크진단**: *장기 외인 vs 단기 외인 비중 추세*, *국내 인기 vs 외인 동향 디커플링*, 지배구조 발표의 실제 이행 검증, *압도적 기술 보유 시 IP 보호·합작 구조*
- **08_이번분기변화 / 09_추적사항**: 페르소나의 ★★★ 적신호·호재가 새로 발생/해소됐는지 우선 점검
- **00_종합진단 마지막**: **"3년 후에도 매력 있는 회사인가? 시총 X조원에 통째로 살 만한가?"** owner-valuation 톤 의무

### 11.1 외인 vs 국내 자금 박스 (조건부 의무)
페르소나의 ★★★ 신호 중 다음이 발견되면 본문에 별도 박스로 강조:
- 외인 비중 추세적 하락 + 국내 인기 상승의 괴리 (적신호)
- 장기 외인 비중 추세적 상승 (호재)
